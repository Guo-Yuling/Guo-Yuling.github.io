Mysql

数据模型

RDBMS

  • 1. 基本定义

    • 类型：关系型数据库（RDBMS）
    • 组成：由多张相互关联的二维表构成，表之间通过字段建立联系。

  • 2. 核心特点

  • 特点 说明
    表结构存储 数据以行和列的形式存储，格式统一，易于维护。
    SQL语言操作 使用标准化SQL（结构化查询语言）进行数据操作，语法通用。

  • 3. 数据表示例

  • 员工表 (employees)
  • id name job dept_id
    1 金庸 总裁 2
    2 张无忌 项目经理 1
    3 杨逍 开发 3
  • 部门表 (departments)
  • id name
    1 研发部
    2 总经办
    3 财务部

  • 4. 表间关系

    • 外键关联：employees.dept_id 字段关联 departments.id，实现数据关联查询。

SQL通用语法

  1. SQL语句可以单行或多行书写，以分号结尾。
  2. SQL语句可以使用空格/缩进来增强语句的可读性。
  3. MySQL数据库的SQL语句不区分大小写，关键字建议使用大写。
  4. 注释：
    • 单行注释：– 注释内容 或 # 注释内容(MySQL特有)
    • 多行注释：/* 注释内容 */
